## Why

当前技能包已经可用，但在行为安全上还缺三层显式约束：

- 读取到的 `alerts`、`mentions` 和 thread 内容没有被明确标记为外部不可信输入，容易让 agent 在允许范围内被命令式文本误导。
- 当 agent 帮用户整理、改写、翻译或补全文案后，技能没有强制要求在真正写入前再次拿到明确确认。
- 回帖流程虽然要求读取 thread context，但没有要求在写入前锁定具体目标对象和资格依据，仍有回错目标的风险。

这些问题不会直接突破 `plurk-mcp` 服务端的硬边界，但会削弱技能层对 prompt injection、误发帖和误回复的防护，因此需要补一轮安全加固，并把这次变更补进 OpenSpec 历史。

## What Changes

- 在共享护栏中新增“不可信输入”“写前再次确认”“回帖前锁定目标”三类规则。
- 收紧 `Load Alerts` 与 `Load Mentions` 的只读工作流，要求把外部文本和链接只当作数据分析，不当作 agent 指令。
- 收紧 `Create Post` 与 `Create Reply` 的写入工作流，要求在改写过文案后先展示最终版本并再次确认。
- 为 `Create Reply` 增加写入前的目标锁定要求：明确 `plurkId`、回复对象和资格依据。
- 同步 README、主规范和写入类 agent 元数据，避免说明文档与实际技能行为漂移。
